aws cli 설치
cmd 그대로 들어가서 명령어 입력하면 사용할 수 있음

aws cli 초기설정
1. access id를 만들어야 함

aws 페이지에 IAM 검색 
-> 좌측 리스트 액세스 관리/사용자 클릭
-> 보안자격 증명 클릭
-> 밑으로 내린 후 액세스 키 만들기 클릭
-> 첫번째 나오는 Command line interface 클릭 후 다음
-> 설명태그 어쩌구는 무시하고 액세스 키 만들기 클릭
-> 액세스 키랑 비밀 액세스 키 나오면 저장해두기

2. cmd에 "aws configure" 입력
순서대로 액세스 키, 비밀 키 입력
세번째 지역은 서울 "ap-northeast-2" 입력
네번째는 "json" 입력

aws 페이지에 s3검색 후 들어가면 여러 버킷(저장소)들을 볼 수 있음

메인은 "hcu-webzine.com" 나머지는 그에 상속되어 있는 버킷들이라 생각해도 좋음
archive.hcu-webzine.com -> 2016~2020 웹진 저장되어 있음
2021.hcu-webzine.com -> 2021 웹진 저장되어 있음

올해는 시간이 없어서 netlify를 사용하지 못했는데,
나중에 2023 웹진 만들때는 netlify 사용하거나, 또 aws 쓸거라면 2022.hcu-webzine.com 버킷만들거나 archive에 넣어서 관리하면 됨.
새로운 버킷 만들어서 페이지 올리는건 pdf 파일 참고.(버킷 초기에 만들때 ALC 활성화하기!)

hcu-webzine.com 기존에 있던 파일들 다 지우면 새로운 파일들로 채우기
작업 -> ACL 활용하여 퍼블릭 엑세스 설정 클릭
파일을 올린다고 바로 사이트가 바뀌는건 아님. s3 버킷의 캐시 메모리를 무효화 해줘서 새로 받아들이게 해야함

이때 aws cli를 사용하는데, 우선 버킷 배포 코드를 받아야 함.
aws에 cloudfront 검색 후 들어가면 대문자 영어로 어쩌구 된 코드들이 있는데, hcu-webzine.com에 해당하는 코드 복사하면 됨.

코드 복사 후, cmd 들어가서 다음 명령어 입력 :
aws cloudfront create-invalidation --distribution-id 배포코드 --path "/*"

위 과정들을 완료하면 아마 페이지가 바뀌어 있을것임.
수정사항이 있을 때는 파일 변경후, 퍼블릭 설정, cloudfront무효화 다시 해주면 됨.
